- 依据：12 个 Academy P0 视频，2 秒抽帧、Whisper medium 转写、逐步骤证据表。
- 对齐目标：补齐 `work/AniShort-like-PRD-technical-design.md` 中标准工作流、节点规格、3D、全景、团队/个人和验收依据不足的部分。

## 1. 证据优先规则

开发过程中若 PRD、WBS、设计稿或实现细节出现逻辑不清、描述冲突、选项缺失、模型列表不确定、页面状态不确定，开发者必须先使用 Chrome 插件和 CDP 检查 AniShort 原站当前实现，再补充实现方案。不得靠自行猜测补齐实际业务逻辑。复核后需要以新的时间戳文件记录证据、截图、接口行为和对 PRD/WBS 的影响。

## 2. 核心工作流修订

### 2.1 标准短剧工作流

标准链路修订为：

```text
剧本节点 -> 剧本优化/分段 -> 主体提取 -> 主体生成 -> 主体库 -> 分镜生成 -> 分镜图生成 -> 视频生成 -> 剪辑预览 -> 成片审阅
```

关键规则：
- 剧本优化/分段是新版工作流的前置步骤，不应默认跳过。
- 主体提取只负责冻结主体资产表，主体生成负责生成主体提示词和设定图。
- 分镜生成之前应优先拉取主体库，避免分镜中出现找不到或对不上的主体。
- 九宫格、四宫格、剧情故事版都属于分镜图生视频路径；熟练用户也可走自由式视频生成路径。

### 2.2 自由式短剧工作流

自由式工作流以视频节点可执行为核心，不强制从剧本一路生成。用户可直接组织提示词、参考图、参考视频、主体引用、历史素材和上传素材，快速生成视频。PRD 需要把它作为标准流之外的正式入口，而不是异常路径。

## 4. 剧本分段节点详细规则

- 子能力一：剧本整理。整理格式、清洗多集剧本、处理导入文本。
- 子能力二：剧本审阅。检查情节、钩子、冲突、角色出现是否合理。
- 子能力三：分镜段落。把剧情拆为适合图像/视频生产的段落。
- 输出模式：原样输出时不得改动原剧本内容；AI 改写时需要保留原文/优化稿对照。
- 分段模式：智能分段根据剧情和对白密度自动拆；固定分段按用户设置的段数或时长拆。
- 时长规则：段落一般应落在 1-15 秒区间；实际分段时长需要和目标时长对比，并提示过短/过长。

## 5. 主体提取与主体生成详细规则

- 主体提取输出主体资产表，不直接生成设定图。
- 资产表至少包含：主体名称、分类、剧中描述、是否已有主体库匹配、是否启用、是否需要生成、备注。
- 用户可在主体提取阶段增删主体，修正名称和分类。
- 主体生成根据资产表生成每个主体的提示词，允许针对单个主体重新生成。
- 角色、场景、道具的生成入口需要分开，且道具缺失时应给出可读错误或跳过说明。
- 主体生成图必须由用户手动分类加入主体库；系统不应静默自动入库。
- 主体库命名不允许空格和特殊符号；分类错误会影响后续可用功能。
- 主体库改名只影响库内条目，不自动回写既有画布节点。
- 上传素材可以替换主体，更新后下游节点需要展示主体库已更新提示。

## 6. 分镜生成详细规则

- 分镜生成节点由剧本节点或剧本分段节点上方命令创建。
- 参数包括分段方式、目标时长、分段数、九宫格/四宫格/剧情故事版模式。
- 2 分钟拆 8 段时，每个九宫格对应 15 秒，这是时长换算的典型验收案例。
- 输出内容包含主体汇总表、九宫格/四宫格图片提示词、视频提示词。
- 生成分镜图前，需要从主体库拉取主体设定图；若检测到缺失主体，允许自动补充生成。
- 批量生成分镜图会创建多个图片节点，并自动创建名为九宫格分镜组的分组。
- 分镜图可调整提示词后重新生成，历史记录必须可回看和复用。

## 7. 3D 导演台详细规则

- 3D 导演台不是节点属性面板，而是从节点按钮进入的独立全屏编辑器。
- 页面布局包括中间 3D 视口、底部角色/人物资源条、左侧 3D 世界/对象列表、右侧属性面板。
- 测量缩放流程：选择 3D 世界 -> 点击测量缩放 -> 开启捕捉 -> 点击已知物体两端 -> 输入真实长度 -> 更新世界比例。
- 设置轴心流程：选择 3D 世界 -> 点击设置轴心 -> 在剧情发生区域点击地面 -> 应用 -> 归零。
- 应用归零是应用和归零的快捷合并，不代表改变场景内容本身，而是重置坐标来改变新角色默认生成位置。

## 8. 全景生成详细规则

- 图片节点上方的全景命令可以创建全景生成节点；底部节点库也能创建。
- 全景节点支持参考图生成和文本生成。
- 输入区包括参考图、提示词、分辨率；提示词可以为空。
- 生成后支持全屏预览、全景模式拖拽查看、当前视角截图。
- 自动输出支持四视图和十二视图，并按输出批次自动分组。

## 9. 个人版与团队版规则

- 同一账号可拥有个人空间和多个团队空间。
- 个人积分与团队积分分开充值、分开消耗、分开管理。
- 个人项目可分享链接，但接收者必须申请，所有者在协作管理批准后才能进入。
- 团队项目没有外部协作链接，只允许团队成员查看和编辑。
- 团队创建者可创建/管理多个团队，成员可加入多个团队，创建者和成员都能创建团队项目。
- 团队管理页需要展示团队列表、成员列表、余额；只有创建者能充值和管理额度。
- 邀请成员通过链接发起，加入申请通过通知处理，创建者批准后可分配积分或周期额度。
- 图片和视频由谁生成，就存储在谁的个人账号下；存储超额时生成和导出可能失败。

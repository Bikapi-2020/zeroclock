## 1. 全局执行规则

### 1.1 证据优先门禁

开发中遇到以下情况，必须暂停实现并复核原站：

- PRD/WBS 对功能描述不清。
- 节点选项、按钮、模型列表不确定。
- Academy 视频与当前原站页面不一致。
- 计费、权限、存储归属存在歧义。
- 3D/全景等交互行为无法从文档判断。

复核要求：

1. 使用 Chrome 插件观察 UI。
2. 使用 CDP 检查 DOM、Network、Console、Local Storage、接口 payload。
3. 记录截图、接口、状态变化。
4. 生成新时间戳证据文件。
5. 更新 PRD/WBS 对应章节后再开发。

### 1.2 阶段 Definition of Done 补强

每个阶段除原 WBS DoD 外，新增：

- 所有新增/变更 nodeId 已同步 PRD 6.3 和 WBS 8.3。
- 每个核心流程至少有一个自动化 E2E。
- 每个含模型任务的功能都有任务状态机和失败态测试。
- 每个涉及积分/存储的任务都有扣费、失败、不扣费边界测试。
- Academy 证据引用已进入验收报告。
- 不清晰逻辑已完成 Chrome/CDP 复核，或明确列为阻断项。

## 2. Phase 2：画布内核补强

### 2.1 开发任务

#### 2.1.1 版本历史

- 实现画布自动快照。
- 支持手动创建版本。
- 支持版本列表。
- 支持恢复版本。
- 版本恢复包含节点、连线、分组、资源引用、节点隐藏状态、连线显示状态、视口位置和缩放。

数据任务：

- `canvas_versions` 表。
- `canvas_snapshots` 对象存储或压缩 JSON。
- 快照差异压缩策略。
- 恢复前自动生成当前版本备份。

#### 2.1.2 节点列表

- 节点搜索。
- 节点定位。
- 节点隐藏/显示。
- 节点分组显示。
- 节点数量统计。
- 空态。

#### 2.1.3 连线显示控制

- 底部工具条显示/隐藏连线。
- 隐藏连线不删除边。
- 隐藏连线后节点端口隐藏。
- 恢复连线后端口可编辑。

#### 2.1.4 资源管理器基础

- 当前画布图片/视频/音频列表。
- 其他画布素材切换。
- 全屏预览。
- 缩略图大小滑块。
- 历史生成版本。
- 查看/复制提示词。
- 定位原节点。
- 从历史创建新节点。

### 2.2 单元/组件测试

| 测试项 | 断言 |
| --- | --- |
| 快照序列化 | 节点、边、分组、资源引用均被保存 |
| 快照恢复 | 恢复后数据 hash 与快照一致 |
| 节点搜索 | 按标题、类型、创建者可搜索 |
| 节点隐藏 | hidden 状态持久化 |
| 连线隐藏 | edge 数据不删除，仅 UI 不显示 |
| 资源历史 | 同一节点多版本资源按时间排序 |
| 提示词复制 | 历史 prompt 可读取 |

### 2.3 集成/E2E 测试

| E2E | 步骤 | 验收 |
| --- | --- | --- |
| 版本恢复 | 创建 5 个节点 -> 生成版本 -> 删除节点 -> 恢复版本 | 节点、连线、资源、视口恢复 |
| 节点定位 | 创建远距离节点 -> 搜索 -> 点击定位 | 视口移动到节点区域 |
| 大画布隐藏连线 | 创建 1000 节点 3000 连线 -> 隐藏连线 -> 拖动画布 | 交互不卡死，连线恢复后可编辑 |

### 2.4 阶段验收标准

- 版本恢复不丢节点/连线/资源。
- 节点列表能支撑复杂画布导航。
- 隐藏连线显著改善大画布可读性。
- 资源管理器能回看历史提示词并复用旧素材。
- 与 P0-3 `画布工作区、节点介绍` 关键流程一致。

## 3. Phase 3：节点系统骨架补强

### 3.1 开发任务

#### 3.1.1 节点创建入口统一

实现以下入口：

- 底部添加节点按钮。
- 画布空白处右键。
- 快捷键。
- 节点命令栏派生。
- 图片节点“全景”命令派生。

统一要求：

- 相同 nodeId 默认数据一致。
- 派生入口自动携带上游引用。
- 权限只读时禁止创建，允许查看/复制。

#### 3.1.2 节点命令栏状态机

适用节点：

- `input-node`
- `story-optimize-node`
- `subject-extract-lite-node`
- `setting-gen-node`
- `storyboard-gen-node`
- `output-node`
- `video-node`
- `panorama-gen-node`
- `scene3d-director-node`

命令状态：

```ts
type NodeCommandState =
  | "hidden"
  | "disabled"
  | "ready"
  | "running"
  | "completed"
  | "failed";
```

#### 3.1.3 节点默认状态

每个节点必须具备：

- 空态。
- 配置态。
- 执行态。
- 成功态。
- 失败态。
- 历史态。
- 只读态。

### 3.2 节点级开发/测试/验收矩阵补强

### 3.3 阶段验收标准

- 所有节点均可通过节点库创建。
- 所有节点命令栏具有依赖状态。
- 派生节点自动携带上游引用。
- 只读模式禁止创建和执行。
- 与 P0-3 节点介绍中的命令栏顺序一致。

## 4. Phase 4：剧本、主体与分镜工作流补强

### 4.1 开发任务拆解

#### 4.1.1 剧本节点

- 粘贴剧本。
- 上传文档。
- AI 剧本创作。
- 多集拆分。
- 原始剧本版本保存。
- 全屏预览。
- 命令进入剧本分段。

#### 4.1.2 剧本分段节点

- 子能力：剧本整理。
- 子能力：剧本审阅。
- 子能力：分镜段落。
- 原样输出。
- AI 改写。
- 智能分段。
- 固定分段。
- 目标总时长。
- 分段数量。
- 实际分段时长估算。
- 时长偏差提示。
- 原文/优化稿对照。

#### 4.1.3 主体提取节点

- 从剧本分段提取角色、场景、道具。
- 输出主体资产表。
- 主体表增删改。
- 主体启用/禁用。
- 主体名称校验。
- 主体分类校验。
- 主体库匹配已有/缺失。
- 进入主体生成。

#### 4.1.4 主体生成节点

- 读取主体资产表。
- 过滤已禁用主体。
- 过滤主体库已存在且无需重生成主体。
- 为每个主体生成提示词。
- 单主体重生成提示词。
- 角色设定图生成。
- 场景设定图生成。
- 道具设定图生成。
- 单主体图片重试。
- 手动分类入库。
- 上传素材替换主体。
- 主体库更新提示。

#### 4.1.5 主体库

- 角色/场景/道具分类。
- 名称校验：不能为空、无空格、无特殊符号。
- 去重。
- 别名。
- 版本历史。
- 手动入库。
- 改名不回写画布节点。

#### 4.1.6 分镜生成节点

- 分段方式选择。
- 影片总时长设置。
- 分段数设置。
- 四宫格/九宫格/剧情故事版模式。
- 主体汇总表输出。
- 图片提示词输出。
- 视频提示词输出。
- 全屏查看。
- 从主体库拉取主体。
- 缺失主体检查。
- 自动补主体。
- 批量生成分镜图。
- 自动创建九宫格分镜组。
- 修改提示词后重生成。

### 4.4 E2E 验收路径

#### 4.4.1 原样输出路径

1. 创建项目。
2. 粘贴一集剧本。
3. 点击剧本分段。
4. 选择原样输出。
5. 选择智能分段。
6. 设置目标总时长。
7. 执行分镜段落。
8. 对比输出文本和输入文本。

验收：

- 原样输出未改动原文本。
- 生成段落有时长估算。
- 时长偏差有提示。

#### 4.4.2 主体资产表路径

1. 从分段结果点击主体提取。
2. 等待主体资产表生成。
3. 手动新增一个道具。
4. 删除一个无关主体。
5. 修改一个角色名称。
6. 执行主体库校验。
7. 进入主体生成。

验收：

- 用户修改全部保留。
- 名称校验生效。
- 已有/缺失主体标记正确。

#### 4.4.3 主体生成路径

1. 读取主体资产表。
2. 对缺失主体生成提示词。
3. 单独重生成一个角色提示词。
4. 生成角色、场景、道具设定图。
5. 将设定图手动加入主体库。

验收：

- 已有主体不重复生成。
- 单主体失败可重试。
- 道具为空时有可读提示。
- 入库需要手动确认。

#### 4.4.4 分镜路径

1. 执行分镜生成。
2. 设置总时长 2 分钟。
3. 设置分段数 8。
4. 选择九宫格。
5. 查看主体汇总、图片提示词、视频提示词。
6. 拉取主体库。
7. 批量生成分镜图。
8. 检查九宫格分镜组。

验收：

- 每个九宫格时长为 15 秒。
- 输出三类内容完整。
- 图片节点自动创建。
- 分组成员正确。

### 4.5 阶段退出条件

- 新版标准链路可跑通。
- 主体提取和主体生成已拆分。
- 分镜输出能驱动图像和视频。
- 主体库一致性规则生效。
- 与 P0-2、P0-7、P0-8、P0-9 证据一致。
